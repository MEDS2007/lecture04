# CSS ANIMATION: **1** ## COMPONENTS - [**transitions**](#/2/3) (things to change, over *time*) - [**transformations**](#/2/4) (shape-shifting, *no* timing) - [**animation**](#/2/6) (composed *timeline* keyframes) You can use these separately or together.
# CSS ANIMATION: **2** **TRANSITION:** EXAMPLE 1 - transition goes in the **element‘s style** (the `div`) - things to change go in the element‘s `:hover` style ```css div { width: 100px; background: red; transition: all 2s; } div:hover { width: 400px; background: blue; } ``` [W3Schools: CSS3 transition Property](http://www.w3schools.com/cssref/css3_pr_transition.asp)
# CSS ANIMATION: **3** **TRANSITION:** EXAMPLE 2 The **trigger** can be **something other** than :hover: ```css input[type=text] { width: 100px; transition: all .35s ease-in-out; } input[type=text]:focus { width: 250px; border: 2px solid #0f0; } ``` [Early article on `transition`, Dan Cederholm, 2010](http://alistapart.com/article/understanding-css3-transitions)
# CSS ANIMATION: **4** **TRANSITION** CONTROLS Changes CSS properties over a **given time** - three examples: ```css transition: all 2s; transition: color 2s 1s; /* add 1s delay */ transition: color, width 2s ease-in-out 1s; ``` - **property**: the CSS to transition (comma-separated, all) - **duration**: transition length (secs/m'secs `s`|`ms`, `infinite`) - **timing-function**: (optional) ‘speed curve’ of the transition - **delay**: (optional) time before transition starts [timing-function easing examples and demo](http://easings.net/)
# CSS ANIMATION: **5** **TRANSFORM** ```css transform: rotate(70deg); ``` Can be used alone (not animated) or with `transition` on an element, then `transform` on :hover/another action. The **values** of the `transform` property are: `translate(540px,-200px);` x, y `rotate(360deg);` `scale(2, 0.5);` x, y `skew(10deg);` x axis `matrix(1, -0.3, 0, 1, 0, 0)` scaleX, skewY, skewX, scaleY, translateX, translateY
# CSS ANIMATION: **6** **TRANSFORM** REFERENCES, TUTORIALS - [W3Schools reference: CSS3 transform Property](http://www.w3schools.com/cssref/css3_pr_transform.asp) - [W3Schools tutorial: CSS3 2D Transforms](http://www.w3schools.com/css/css3_2dtransforms.asp) - [W3Schools: CSS3 3D Transforms](http://www.w3schools.com/css/css3_3dtransforms.asp) - [W3Schools: CSS3 Perspective](https://www.w3schools.com/CSSref/css3_pr_perspective.asp)
# CSS ANIMATION: **7** **ANIMATION** The [CSS `animation` property](http://www.w3schools.com/cssref/css3_pr_animation.asp) binds a custom `animation` to an element identified by it’s `selector` e.g. ```css .see-this { animation: change-stuff 5s infinite; } ``` (animation name, duration, how many times to run) `infinite` runs forever, seconds/milliseconds limit time The changes are user-defined in a separate `@keyframes` code block…
# CSS ANIMATION: **8** **ANIMATION** KEYFRAMES ```css @keyframes change-stuff { 25% { styles for 1/4 way through; } 50% { styles for 1/2 way through; } 75% { styles for 3/4 way through; } } ``` Download the [code for this demo animation](https://github.com/CTEC3905/css-animation-01) or [view it here](https://ctec3905.github.io/08-lab/) (See [W3Schools CSS3 Animations](http://www.w3schools.com/css/css3_animations.asp))
# CSS ANIMATION: **9** **ANIMATION** WHAT IS (NOT) ANIMATABLE? **not** `display: none` or `display: block`, **not** the property value `auto` (e.g. for height) etc. [Animatable CSS properties (W3Schools)](http://www.w3schools.com/CSSref/css_animatable.asp) triggered by focus/hover/etc.… that can be styled e.g.: `:focus :blur :hover :active input[type=checkbox]:checked …`

# JAVASCRIPT **INTRO** 02 ## How is JavaScript used? - form **validation** and **processing** of user input - handling **data** (e.g. JSON) - interactive **user interfaces** and feedback - web-based **apps** (mobile and desktop) - web and mobile **frameworks** - **applications** built on web technologies - **games** with HTML5 canvas ([HTML5 Games website](http://html5games.com/Best/702b9531-c136-437a-ab97-0b209d893b55)) - **[3D animations](https://tutorialzine.com/2013/09/20-impressive-examples-for-learning-webgl-with-three-js)** with canvas and WebGL e.g. [cars](http://carvisualizer.plus360degrees.com/threejs/), [Aquarium](http://webglsamples.org/aquarium/aquarium.html) Some of these use the JavaScript library [Three.js](https://threejs.org/)
# JAVASCRIPT **INTRO** 03 ## THE BROWSER EVENT LOOP In a browser, everything runs in an **event loop** so you use **event handlers** to **listen** for **events** triggered by the **user** or other **processes** e.g. ``` javascript element.addEventListener("click", do_stuff); ``` This **listens** for a **click** on a specific HTML **element**, then calls a `function`—in this case called `do_stuff`—which does whatever you code it to do. While **CSS** can be triggered by focus/hover/etc., **JavaScript** can listen for **hundreds** of events. See: [Web Event reference (MDN)](https://developer.mozilla.org/en-US/docs/Web/Events)
# JAVASCRIPT/THE DOM: **01** When you open any webpage in a browser, its **HTML** is **loaded** and **rendered visually** on the screen To do this, the browser builds the **Document Object Model** (DOM) of that page, which is a **model of its logical structure**
# JAVASCRIPT/THE DOM: **02** The **DOM** represents a document as a **tree structure** where HTML elements are **interrelated nodes** in the tree, and All nodes in the tree have **relationships** with each other: - nodes all have **parent** nodes, - nodes can have **child** nodes, - nodes at the same level are **siblings**
# JAVASCRIPT/THE DOM: **03** The DOM can be represented as a **nested set of boxes**: ![](images/dom.jpg) JavaScript can be used to **alter the DOM** of a page dynamically… to **add**, **delete** and **modify** elements.
# JAVASCRIPT/THE DOM: **04** For example, in the structure illustrated above: - `html` has two **children** (`head` and `body`) - `head` has one **child** (`title`) and one parent (`html`) - `title` has one **parent** (`head`) and **no children** - `body` has two **children** (`h1` and `p`) and **one parent** (`html`) - `p` and `h1` are **siblings** under their parent **body** - `title`, `p` and `h1` also contain "text nodes" Understanding the **relationships between elements** in an HTML document enables us to **manipulate them** with **JavaScript**
# JAVASCRIPT/THE DOM: **05** There is a predefined `document` object in JavaScript, which can be used to **access all elements** in the **DOM**. This `document` object is the owner (or **root**) of **all objects in your webpage**. So to access objects in an HTML page, you always start with the `document` object, e.g.: ```javascript document.body.innerHTML = "Some text"; ``` `body` is an element of the DOM so you access it using the `document` object, then you can change the content of its `innerHTML` property.
# JAVASCRIPT DEMOS: **01** The simplest possible example: In the **HTML** (.html) file: ```html <p id="my-id">Click me!</p> ``` In the **JavaScript** (.js) file: ``` javascript // store a reference to the element to be clicked: const clickThing = document.getElementById("my-id"); // what to do when the user clicks: function do_stuff() { alert("You clicked me!"); } // or you can use the simpler "fat arrow" syntax: const do_stuff = () => { alert("You clicked me!"); } // "listen" for the click: clickThing.addEventListener("click", do_stuff); ```

# JAVASCRIPT OBJECTS: **01** A [JavaScript object](http://www.w3schools.com/js/js_objects.asp) contains `name:value` pairs inside curly braces `{}` (there’s **no comma** after the last item!). Here, the `range` object is a child of he `car` object: ```javascript const car = { manufacturer: "Tesla", model: "Model S", range: { normal: "360", ludicrous: "200" }, colour: "red", start: function() {startMyCarStuffHere} }; car.range.ludicrous; // returns: 200 ``` An **array of objects** is a common **JSON data format** used for exchanging data on the web e.g. from **APIs**
