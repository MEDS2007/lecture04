## CSS ANIMATION: **2** **TRANSITION EXAMPLE 1**: change **colour** and **width** on `:hover` - the **transition rules** go in the element’s *style* (e.g. a `div`) - things **to change** go in the element’s *`:hover` style* ```css div { width: 100px; background: red; transition: all 2s; } div:hover { width: 400px; background: blue; } ``` [W3Schools: CSS3 transition Property](http://www.w3schools.com/cssref/css3_pr_transition.asp)
## CSS ANIMATION: **3** **TRANSITION EXAMPLE 2**: **highlight an input field** on `:focus` (hover is not the only **trigger**) ```css input[type=text] { background: #eee; /* pale grey */ border-color: black; transition: all .35s ease-in-out; } input[type=text]:focus { background: white; border-color: green; border: 2px solid #0f0; } ``` [Early article on `transition`, Dan Cederholm, 2010](http://alistapart.com/article/understanding-css3-transitions) (**delete lines** that start with `-webkit-` as it’s no longer needed)
## CSS ANIMATION: **4** **TRANSITION** CONTROLS Changes CSS properties over a **given time** - three examples: ```css transition: all 2s; /* takes 2 seconds */ transition: all 2s 1s; /* add a 1 second delay */ transition: color, width 2s ease-in-out 1s; /* specific */ ``` The **third example** above uses code for the: - **property**: (comma-separated or `all`) is the CSS to transition - **duration**: the transition length (secs/millisecs `s`|`ms`) - **timing-function**: (optional) ‘speed curve’ of the transition - **delay**: (optional) delay before the transition starts [timing-function easing examples (hover over, click for code)](http://easings.net/)
## CSS ANIMATION: **5** **TRANSFORM** ```css transform: rotate(90deg); /* sideways, can be minus */ ``` To animate, use `transition` on an **element** then `transform` on `:hover` or another user action. **Other values** of the `transform:` property are: `translate(540px,-200px);` x, y `scale(2, 0.5);` x, y `skew(10deg, 20deg);` x, y or **all three together**: `matrix(1, -0.3, 0, 1, 0, 0);` (scaleX, skewY, skewX, scaleY, translateX, translateY)

## TYPOGRAPHY CODE **02** how how many `h1` (main heading) tags can you have? - technically, you can have one `h1` tag per **sectioning root** or content section… - …if those *sectioning root elements* (below) carry **self-contained content**: `<article> <section> <aside> <nav>` However, **in practice** (unless you’re a news site) it’s usually best to have: - one `h1` to headline the **whole page** before/first inside the `<main>` tag - `h2` headings in **content sections** that introduce **that section** - **subheadings** inside sections should **descend in order**, starting with `h3` This creates a **semantically structured document outline**. (Adapted from [The Truth About Multiple h1 Tags in the HTML5 Era](https://webdesign.tutsplus.com/articles/the-truth-about-multiple-h1-tags-in-the-html5-era--webdesign-16824))

## TYPOGRAPHY CODE **05** **only a few fonts** are on every computer (see [Typetester](http://classic.typetester.org/)). A font on your computer may look fine in your browser **but** your website users **may not have that font**. > “It **looks fine** on **my computer**” —*well-known moan* of novice web designers! [Google fonts](https://fonts.google.com/) (or web fonts on you own site) allow you to **expand the range** of available **web typefaces** with the CSS property `@font-face`. Best practice is to use them **only for headings** as fonts can take **awhile to load** on **slow connections**, causing a sudden jump from the default font.
## TYPOGRAPHY CODE **06** To use a specific font **local to your website** (and not depend on a Google fonts), you also use `@font-face` ```css @font-face { font-family: Chunkfive; /* name your font and… */ src: url('myfonts/Chunkfive.otf'); /* tell the browser where it is */ } /* a "font stack" offers fall-backs in case the first one fails: */ font-family: Chunkfive, Georgia, Palatino, Times, serif; ``` The *name* gives the font an **identifier** for the **rest of your CSS** that points to the **original font file** when its name is used anywhere in your **CSS stylesheet**. **Best format**: WOFF (OpenType or TrueType with compression and additional metadata) is the [most widely supported](https://www.w3schools.com/css/css3_fonts.asp) but OTF/TTF are also good. - [Web Font Generator (FontSquirrel)](https://www.fontsquirrel.com/tools/webfont-generator) - [css @font-face generator (careful: more options!)](https://transfonter.org/)
## TYPOGRAPHY CODE **07** type in CSS can be **sized** in **different measures**, most commonly: `em`, `%`, and `px` (ems, percentage and pixels). ```css font-size: 1.5em; font-size: 150%; font-size: 12px; ``` `em`s are easiest to understand—`1em` is "normal" size, so `.5em` is half and `2em` is twice normal size `em`s are adjusted according to their **parent element**, and this can produce **confusing size variations**, so…
## TYPOGRAPHY CODE **08** …the `rem` allows for **absolute size adjustments** from a **root em** `font-size`, typically a base `em` size set on the `body` tag. `rem` sizes ignore the `font-size` styles for any containing element, and are always based on the **root** `font-size`: ```css body { font-size: 1em; } /* set the root size in ems */ h1 { font-size: 2rem; } /* always twice root size */ small { font-size: .75rem; } /* always 3/4 root size */ ``` [Rems, ems, and pixels. What’s the difference? (Abbey Fitzgerald 2015)](https://getflywheel.com/layout/rems-ems-and-pixels-whats-the-difference/)
## TYPOGRAPHY CODE **9** **Form fields are different** The default font for **form fields** in browsers is usually **smaller** and in a **different font** than the body text, but you can usually make these default fonts the same using `inherit`: ```css input, textarea { font-size: inherit; font-family: inherit; } ``` Or set the **font-size manually** (and if necessary the **font-family**)

## TYPOGRAPHY CODE **11** you can highlight the entire [::first-line](https://www.w3schools.com/cssref/sel_firstline.asp) of a block of text: ```css blockquote::first-line, /* apply this style to */ p::first-line { /* blockquotes and paragraphs */ font-size: 1.25rem; } ``` The above makes the first line of text in all **blockquotes** and **paragraphs** .25 larger

## JAVASCRIPT **INTRO** 03 ## THE BROWSER EVENT LOOP In a browser, everything runs in an **event loop** so you use **event handlers** to **listen** for **events** triggered by the **user** or other **processes** e.g. ``` javascript element.addEventListener("click", do_stuff); ``` This **listens** for a **click** on a specific HTML **element**, then calls a `function`—in this case called `do_stuff`—which does whatever you code it to do. While **CSS** can be triggered by focus/hover/etc., **JavaScript** can listen for **hundreds** of events. See: [Web Event reference (MDN)](https://developer.mozilla.org/en-US/docs/Web/Events)
## JAVASCRIPT/THE DOM: **01** When you open any webpage in a browser, its **HTML** is **loaded** and **rendered visually** on the screen To do this, the browser builds a **Document Object Model** (DOM) of the HTML tags for the page—a **model of its logical structure**
## JAVASCRIPT/THE DOM: **02** The **DOM** represents a document as a **tree structure** where HTML elements are **interrelated nodes** in the tree, and All nodes in the tree have **relationships** with each other: - nodes all have **parent** nodes, - nodes can have **child** nodes, - nodes at the same level are **siblings**
## JAVASCRIPT/THE DOM: **03** The DOM can be represented as a **nested set of boxes**: ![](images/dom.jpg) JavaScript can be used to **alter the DOM** of a page dynamically… to **add**, **delete** and **modify** elements.
## JAVASCRIPT/THE DOM: **04** For example, in the structure illustrated above: - `html` has two **children** (`head` and `body`) - `head` has one **child** (`title`) and one parent (`html`) - `title` has one **parent** (`head`) and **no children** - `body` has two **children** (`h1` and `p`) and **one parent** (`html`) - `p` and `h1` are **siblings** under their parent **body** - `title`, `p` and `h1` also contain "text nodes" Understanding the **relationships between elements** in an HTML document enables us to **manipulate them** with **JavaScript**
## JAVASCRIPT/THE DOM: **05** There is a predefined `document` object in JavaScript, which can be used to **access all elements** in the **DOM**. This `document` object is the owner (or **root**) of **all objects in your webpage**. So to access objects in an HTML page, you always start with the `document` object, e.g.: ```javascript document.body.innerHTML = "<h1>A heading<h1>"; ``` `body` is an element of the DOM—you access it using the `document` object, then you can change its `innerText or` `innerHTML` property.
## JAVASCRIPT DEMOS: **01** The simplest possible example: In the **HTML** (.html) file: ```html <p id="my-id">Click me!</p> ``` In the **JavaScript** (.js) file: ``` javascript // store a reference to the element to be clicked: const clickThing = document.getElementById("my-id"); // make a function that says what to do when the user clicks: function do_stuff() { alert("You clicked me!"); } // OR use the simpler but more cryptic "fat arrow" function syntax: const do_stuff = () => { alert("You clicked me!"); } // "listen" for the click: clickThing.addEventListener("click", do_stuff); ```
