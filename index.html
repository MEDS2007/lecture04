# CSS ANIMATION: **1** ## COMPONENTS - [**transitions**](#/2/3) (things to change, over *time*) - [**transformations**](#/2/4) (shape-shifting, *no* timing) - [**animation**](#/2/6) (composed *timeline* keyframes) You can use these separately or together.
# CSS ANIMATION: **2** ## **TRANSITION:** EXAMPLE 1 - transition goes in the **element‘s style** (the `div`) - things to change go in the element‘s `:hover` style ```css div { width: 100px; background: red; transition: all 2s; } div:hover { width: 400px; background: blue; } ``` [W3Schools: CSS3 transition Property](http://www.w3schools.com/cssref/css3_pr_transition.asp)
# CSS ANIMATION: **3** ## **TRANSITION:** EXAMPLE 2 The **trigger** can be **something other** than :hover: ```css input[type=text] { width: 100px; transition: all .35s ease-in-out; } input[type=text]:focus { width: 250px; border: 2px solid #0f0; } ``` [Early article on `transition`, Dan Cederholm, 2010](http://alistapart.com/article/understanding-css3-transitions)
# CSS ANIMATION: **4** ## **TRANSITION** Changes CSS properties over a **given time** - three examples: ```css transition: all 2s; transition: color 2s 1s; /* add 1s delay */ transition: color, width 2s ease-in-out 1s; ``` - **property**: the CSS to transition (comma-separated, all) - **duration**: transition length (secs/m'secs `s`|`ms`, `infinite`) - **timing-function**: (optional) ‘speed curve’ of the transition - **delay**: (optional) time before transition starts [timing-function easing examples and demo](http://easings.net/)
# CSS ANIMATION: **5** ## **TRANSFORM** ```css transform: rotate(70deg); ``` Can be used alone (not animated) or with `transition` on an element, then `transform` on :hover/another action. The **values** of the `transform` property are: `translate(540px,-200px);` x, y `rotate(360deg);` `scale(2, 0.5);` x, y `skew(10deg);` x axis `matrix(1, -0.3, 0, 1, 0, 0)` scaleX, skewY, skewX, scaleY, translateX, translateY
# CSS ANIMATION: **6** ## **TRANSFORM** REFERENCES, TUTORIALS - [W3Schools reference: CSS3 transform Property](http://www.w3schools.com/cssref/css3_pr_transform.asp) - [W3Schools tutorial: CSS3 2D Transforms](http://www.w3schools.com/css/css3_2dtransforms.asp) - [W3Schools: CSS3 3D Transforms](http://www.w3schools.com/css/css3_3dtransforms.asp)
# CSS ANIMATION: **7** ## **ANIMATION** The [CSS `animation` property](http://www.w3schools.com/cssref/css3_pr_animation.asp) binds a custom `animation` to an element identified by it’s `selector` e.g. ```css .see-this { animation: change-stuff 5s infinite; } ``` (animation name, duration, how many times to run) `infinite` runs forever, seconds/milliseconds limit time The changes are user-defined in a separate `@keyframes` code block
# CSS ANIMATION: **8** ## **ANIMATION** KEYFRAMES ```css @keyframes change-stuff { 25% { styles for 1/4 way through; } 50% { styles for 1/2 way through; } 75% { styles for 3/4 way through; } } ``` [Download the code for this demo animation] or(https://github.com/CTEC3905/css-animation-01) [view it here](https://ctec3905.github.io/css-animation-01/) (See [W3Schools CSS3 Animations](http://www.w3schools.com/css/css3_animations.asp))

# JAVASCRIPT: **03** ## THE BROWSER EVENT LOOP Inside the browser, everything runs in an **event loop** so you think in terms of **event handlers** that **listen** for events triggered by the user or other processes: ``` javascript myElement.addEventListener("click", do_stuff); ``` While CSS can be triggered by focus/hover/etc., JavaScript can listen for **hundreds** of events, not just ‘click’.
# JAVASCRIPT: **04** ## TWO EXAMPLES In a browser, everything runs in an **event loop** so you use **event handlers** to **listen** for events triggered by the user or other processes: ``` javascript var mystuff = document.getElementById("my-id-here"); mystuff.addEventListener("click", do_stuff); function do_stuff() { alert("You clicked me!"); } ``` While CSS can be triggered by focus/hover/etc., JavaScript can listen for **hundreds** of events: [Web Event reference (MDN)](https://developer.mozilla.org/en-US/docs/Web/Events)

# JAVASCRIPT: **08** ## JAVASCRIPT OBJECTS A [JavaScript object](http://www.w3schools.com/js/js_objects.asp) contains a series of colon-separated name-value pairs (no comma after the last one!!) enclosed in curly braces `{}`: ```javascript var car = { manufacturer: "Tesla", model: "Model S", range: "300", colour: "red", start: function() {startMyCarStuffHere} }; ``` You can have an **array of objects**, which is similar to a common data format called **JSON**, often used for online data sources.
